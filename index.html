Защита от MITM-атак

Минимальные метаданные

Мы собираем абсолютный минимум метаданных. Невозможно отследить, с кем и когда вы общаетесь.

Технологии Mesh

Подробности о том, как мы обеспечиваем вашу безопасность

Сквозное шифрование

Используем Signal Protocol с уникальным ключом для каждого чата. Даже сервер не может прочитать сообщения.

  • Уникальный ключ для каждого чата
  • Сервер не имеет доступа к содержимому
  • Проверка ключей для предотвращения MITM

Forward Secrecy

Ключи сеансов постоянно обновляются, так что компрометация одного ключа не раскрывает прошлые переписки.

  • Постоянная смена ключей
  • Защита прошлых переписок
  • Стойкость к будущим атакам

Анонимность

Tor/I2P для скрытия IP и защиты метаданных — никто не узнает, кто с кем общается и откуда.

  • Скрытие IP-адреса
  • Защита метаданных
  • Обход блокировок

P2P-сеть

Сообщения могут идти напрямую между пользователями без центрального сервера, что исключает единую точку взлома.

  • Децентрализованная архитектура
  • Отсутствие единой точки отказа
  • Прямые соединения между пользователями

Нет логов и привязки

Только случайный SSH ключ, никакой персональной информации не хранится. Нет привязки к телефону или e-mail.

  • Анонимная регистрация
  • SSH-ключ вместо личных данных
  • Нет сбора метаданных

Самоуничтожение данных

Сообщения хранятся зашифрованно и удаляются после доставки, локальные ключи защищены через Secure Enclave/TPM.

  • Автоматическое удаление сообщений
  • Защита локальных данных
  • Использование аппаратной безопасности
